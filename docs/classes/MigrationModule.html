migrateToLatest

  • migrateToLatest(migrationsFolderPath: string): Promise<void>
  • migrateToLatest(allMigrations: Record<string, Migration>): Promise<void>
    • Defined in src/migration/migration.ts:45

    Runs all migrations that have not yet been run.

    The only argument must either be a file path to the folder that contains all migrations OR an object that contains all migrations (not just the ones that need to be run). The keys in the object must be the unique migration names.

    This method goes through all possible migrations (passed as the argument) and runs the ones whose names are alphabetically after the last migration that has been run. If the list of executed migrations doesn't match the list of possible migrations, and error is thrown.

    example
    await db.migration.migrateToLatest(
      path.join(__dirname, 'migrations')
    )
